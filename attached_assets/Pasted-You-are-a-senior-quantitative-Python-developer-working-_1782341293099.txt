Implementation notes:

Each detector must:
Build a candidate pattern from pivots.
Compute a confidence score.
Return a PatternCandidate object (or dict) with pattern_type, confidence, candidate_index, breakout_level, is_channel, etc.
Be confirmed by the existing _confirm_candidate machinery (breakout, channel, trend, or range).
Register the new types in src/backend/similarity_matcher.py:

Update main.py dispatcher so the new types can be emitted by causal_patterns and type_scan modes.
Specific detector logic:

Triple Top/Bottom: find three high/low pivots within tolerance * price_range (or ATR-scaled) of each other, with valleys/peaks between them. Confidence based on peak/trough equality and middle amplitude.
Cup and Handle: detect a U-shaped rounded bottom (cup) followed by a small pullback/consolidation (handle) on the right side. The handle should not retrace more than handle_retrace_max of the cup depth. Confidence based on cup symmetry and handle proportion.
Pennant: detect a strong pole (impulsive move) followed by a small converging triangle. Use the existing triangle logic on the post-pole segment; confidence based on pole strength and convergence.
Rounding Bottom: detect a smooth U-shaped curve using curvature / second-derivative analysis of lows; confidence based on smoothness and depth.
