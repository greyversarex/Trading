Sub-phase 1.3 — Robust levels and trendlines
Create a new class src/backend/level_detector_v2.py named LevelDetectorV2. Keep LevelDetector in level_detector.py unchanged for compatibility, but LevelDetectorV2 should be the default used by new scan modes.

LevelDetectorV2.detect_levels(highs, lows, closes, volumes) must produce:

support_levels: list of horizontal support zones, each with price, strength (0-1), num_touches, first_touch_time, last_touch_time, type = "support".
resistance_levels: analogous.
trendlines: list of diagonal lines, each with slope (price per bar), intercept, type ("support_trendline" or "resistance_trendline"), strength (0-1), touches (list of indices), start_idx, end_idx.
Implementation details:

Pivots: use local fractal extrema from OHLC (same as before but with configurable order).
Horizontal zones: run DBSCAN (sklearn.cluster.DBSCAN) on pivot prices with eps = CONFIG.level.dbscan_eps_factor * volatility_scale. Each cluster is a level; strength is proportional to cluster size, recency, and total volume at touches.
Diagonal lines: for support, fit lines through pairs/triples of low pivots using RANSAC or Theil-Sen regression. Keep the line only if it has ≥ CONFIG.level.min_touches inliers within CONFIG.level.ransac_residual_threshold_factor * volatility_scale. Parallelism for channels is checked via relative slope difference.
Volume profile: build a histogram of close weighted by volume over CONFIG.level.volume_profile_bins. Add POC (point of control) and value area high/low as levels if they are not already covered by DBSCAN clusters.
Round numbers: generate candidate prices near current price with step = CONFIG.level.round_number_step_factor * volatility_scale, rounded to a "nice" number. Add as a level only if it has at least one actual touch within 0.5 * volatility_scale.
Deduplication: merge levels that are within 0.3 * volatility_scale of each other, summing strength.
