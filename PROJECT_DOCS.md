# Сканер Структур Графиков — Документация проекта

## Описание проекта

Система автоматического обнаружения визуально схожих ценовых структур на криптовалютных графиках в реальном времени. Сканирует рынки Binance и находит графики, чья ценовая структура максимально соответствует образцу, предоставленному пользователем в виде скриншота.

### Основные возможности

- Загрузка и обработка скриншотов графиков (OpenCV)
- Извлечение и классификация ценовых структур (20+ типов паттернов)
- Мульти-лейбл классификация — каждый график может содержать несколько паттернов одновременно
- Сканирование рынка Binance в реальном времени (топ-50 пар по объёму)
- Мультитаймфрейм анализ: 1м, 5м, 15м, 1ч, 4ч, 1д
- Скользящее окно (30, 50, 70, 100 свечей) для обнаружения паттернов внутри тренда
- Сопоставление структур с использованием DTW и геометрического сходства
- Детекция 19 свечных паттернов с визуализацией позиций на графике
- Анализ уровней Фибоначчи
- Обратная связь для улучшения точности

## Рабочий процесс разработки

### Общая схема

```
Replit (разработка) → Git репозиторий → VPS сервер (продакшн)
```

1. **Разработка в Replit** — весь код пишется и тестируется в среде Replit
2. **Коммит в репозиторий** — изменения сохраняются в Git
3. **Обновление на сервере** — код подтягивается из репозитория на VPS

### Обновление кода на сервере

После коммита изменений в Replit, на сервере выполнить:

```bash
cd /opt/Trading
source venv/bin/activate
git pull
pkill -9 -f "python run.py"
sleep 3
nohup python run.py > app.log 2>&1 &
sleep 3
tail -10 app.log
```

## Решение типичных проблем

### Сайт не открывается (отказ в соединении)

1. Проверить, запущен ли процесс: `ps aux | grep "python run.py"`
2. Проверить порт: `ss -tlnp | grep 5000`
3. Если процесс не найден — запустить приложение
4. Если порт занят другим процессом — убить его и перезапустить

### Два процесса Python на одном порту

```bash
pkill -9 -f "python run.py"
sleep 3
nohup python run.py > app.log 2>&1 &
```

### Ошибки при запуске

```bash
tail -50 app.log
```
Смотреть логи для определения причины ошибки.

### Обновление кода не применилось

Убедиться, что:
1. Изменения закоммичены в репозиторий
2. На сервере выполнен `git pull`
3. Приложение перезапущено после `git pull`
