# Сканер Структур Графиков

## Overview

Проект представляет собой Python-систему, предназначенную для автоматического обнаружения визуально схожих ценовых структур на криптовалютных графиках в реальном времени. Основная цель — сканирование рынков Binance и поиск графиков, чья ценовая структура максимально соответствует образцу, предоставленному пользователем в виде скриншота, фокусируясь на **структурном визуальном сходстве**.

Ключевые возможности включают:
- Загрузка и обработка скриншотов графиков.
- Извлечение и классификация ценовых структур (пивотные точки, тренды, паттерны).
- Сканирование рынка Binance в реальном времени по множеству торговых пар и таймфреймов.
- Сопоставление структур на основе геометрического и реляционного сходства.
- Механизм обратной связи для улучшения точности системы.

Бизнес-видение проекта — предоставить трейдерам мощный инструмент для быстрого выявления потенциальных торговых возможностей на основе проверенных графических паттернов, автоматизируя процесс визуального анализа и снижая временные затраты на мониторинг рынка.

## System Architecture

Система построена на архитектуре "клиент-сервер" с FastAPI на бэкенде и простым HTML/CSS/JS фронтендом.

**UI/UX Decisions:**
- Фронтенд реализован как Single Page Application (SPA) с использованием Tailwind CSS для стилизации.
- Интерфейс поддерживает русскую локализацию.
- Для отображения графиков и визуализации данных используется TradingView виджет.
- Предусмотрены элементы управления для загрузки скриншотов, управления сканированием, настройки порогов схожести и отправки обратной связи.
- Реализована темная тема UI.

**Technical Implementations & Feature Specifications:**
- **Загрузка и Обработка Скриншотов:** Система принимает скриншоты, автоматически выделяет ценовую линию с помощью OpenCV и удаляет шумы (UI элементы, сетки, оси).
- **Извлечение Структуры:** Используется Zigzag-алгоритм для детекции пивотных точек (локальных максимумов/минимумов). Извлекается относительная геометрия, тренды, и поведенческие паттерны (сжатие, консолидация). Поддерживается классификация 20+ типов структур, включая Двойные вершины/донышки, Голова и плечи, Флаги, Клинья и др. Внедрена оценка уверенности (confidence) для каждого пивота и паттерна. **Мульти-лейбл классификация:** каждый график может содержать несколько обнаруженных паттернов одновременно (поле `detected_patterns`), что исключает потерю вторичных паттернов.
- **Реальное Сканирование Рынка:** Подключение к Binance WebSocket API для мониторинга в реальном времени топ-50 торговых пар по объёму. Анализ проводится на мультитаймфреймах: 1м, 5м, 15м, 1ч, 4ч, 1д. Используются реальные данные Binance klines API, с фаллбэком на симуляцию при недоступности. **Скользящее окно:** помимо полных 100 свечей, анализируются подсегменты (последние 30, 50, 70 свечей) для обнаружения паттернов, формирующихся внутри тренда.
- **Сопоставление Схожести:** Сравнение происходит между структурами (не изображениями), используя геометрическое и реляционное сходство, а также Dynamic Time Warping (DTW) для эластичного сравнения линий. Учитывается зеркальная инвариантность. Результат сопоставления выражается в нормализованной оценке (0-100%) с настраиваемым порогом. **Ослабленный штраф за тип:** используется `detected_patterns` overlap — если два графика имеют общие обнаруженные паттерны, штраф за разную первичную классификацию снижается. Базовые штрафы за несовпадение типов и зеркальные совпадения также смягчены для повышения точности.
- **Анализ Фибоначчи:** Модуль `fibonacci_analyzer.py` для определения и анализа касаний уровней Фибоначчи (23.6%, 38.2%, 50%, 61.8%, 78.6%) и расширений (127.2%, 161.8%, 261.8%), включая детекцию отскоков и оценку качества касаний. Расширения проецируются за пределы свинга (выше для восходящих, ниже для нисходящих трендов).
- **Свечные Паттерны:** Детекция 19 распространенных свечных паттернов (Доджи, Молот, Поглощение и т.д.) с возможностью сканирования рынка по ним. Применяется контекстная фильтрация по предшествующему тренду (бычьи разворотные паттерны валидны только после нисходящего тренда, медвежьи — после восходящего). Используется параметр `max_age` (по умолчанию 10 свечей) для отображения только свежих паттернов.
- **Детекция Свежести Паттернов:** Все структурные паттерны (треугольники, клинья, двойные вершины/донышки, голова-плечи, флаги) проверяются на пробой. Если цена уже вышла за границы паттерна (пробила нэклайн, вышла из треугольника и т.д.), паттерн отмечается как неактивный и исключается из результатов сканирования. Допуск для двойных вершин/донышек ужесточён с 10% до 3%.
- **Обратная Связь Пользователя:** Реализован механизм обратной связи ("Релевантно"/"Нерелевантно") для каждого совпадения для потенциального обучения системы.
- **Обзор Рынка:** Возможность сортировки по изменению цены за 24 часа ("Топ роста 24ч", "Топ падения 24ч") с использованием данных CoinGecko.

**System Design Choices:**
- **Бэкенд:** Python с использованием FastAPI для асинхронной обработки запросов и WebSocket для обновлений в реальном времени.
- **Обработка Изображений:** OpenCV для анализа скриншотов.
- **Математические Вычисления:** NumPy, SciPy, scikit-learn для обработки данных и машинного обучения.
- **База Данных:** SQLite для хранения данных (через `aiosqlite`).
- **Модульная Структура:** Код организован в модули (`image_processor.py`, `structure_extractor.py`, `similarity_matcher.py`, `binance_scanner.py`, `candle_patterns.py`, `fibonacci_analyzer.py`) для лучшей поддерживаемости.

## External Dependencies

- **Binance WebSocket API:** Для получения рыночных данных в реальном времени.
- **Binance klines API:** Для получения исторических данных свечей.
- **CoinGecko API:** Для получения информации о рыночной капитализации, изменении цены за 24 часа и списка торговых пар.
- **TradingView Widgets:** Для интеграции интерактивных графиков в пользовательский интерфейс.
